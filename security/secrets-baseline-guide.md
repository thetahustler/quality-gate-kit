# Secrets Baseline & Gitignore Hygiene

Guidance on two common pitfalls with secret detection and gitignore patterns.

---

## 1. Commit `.secrets.baseline` — Do NOT Gitignore It

The `.secrets.baseline` file (generated by `detect-secrets`) **must be committed to the repo**. It is NOT a secret — it is a record of known false positives that the team has reviewed and approved.

### Why

- `detect-secrets` compares new scan results against the baseline. Without a committed baseline, the pre-commit hook cannot distinguish known false positives from real leaks.
- If `.secrets.baseline` is gitignored, every developer gets a fresh scan with no context, leading to noisy false positives and eventually `--no-verify` bypasses.
- The baseline acts as an audit trail: each entry was reviewed by a human and marked safe.

### How to Set Up

```bash
# Generate initial baseline
detect-secrets scan > .secrets.baseline

# Audit it (review each finding, mark as real or false positive)
detect-secrets audit .secrets.baseline

# Commit it
git add .secrets.baseline
git commit -m "chore: add secrets baseline"
```

### Common Mistake

Adding `.secrets.baseline` to `.gitignore` because it "looks like a secrets file." It is not — it contains hashes and metadata, not actual secrets.

---

## 2. Avoid Overly Broad `*token*` Gitignore Patterns

### The Problem

A `.gitignore` entry like:

```
*token*
```

Matches **any file or directory** with "token" in the name. This silently ignores:

- `src/utils/token_refresh.py` — your OAuth refresh logic
- `tests/test_token_validation.py` — your test file
- `docs/token-flow.md` — your documentation
- `config/tokenizer.json` — your NLP config

You will not get a warning. Git simply won't track these files, and you may not notice until production breaks.

### The Fix

Be specific about what you are ignoring:

```gitignore
# BAD: catches any file with "token" in the name
*token*

# GOOD: ignore specific token files
.token
*.token
token.json
token.pickle
```

Or use directory-scoped patterns:

```gitignore
# Only ignore token files in the root directory
/token.json
/.token
```

### How to Check

If you suspect files are being silently ignored:

```bash
# List all ignored files that match a pattern
git status --ignored | grep -i token

# Check if a specific file is ignored
git check-ignore -v src/utils/token_refresh.py
```

---

## Checklist

- [ ] `.secrets.baseline` is committed (not in `.gitignore`)
- [ ] No `*token*` or `*secret*` wildcard patterns in `.gitignore`
- [ ] Gitignore patterns are specific to actual credential files, not broad globs
- [ ] `detect-secrets audit .secrets.baseline` has been run at least once
